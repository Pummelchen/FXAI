FXAI Reference
==============

Executive Summary
-----------------
FXAI is a native MetaTrader 5 framework for building, testing, and operating cost-aware FX prediction models inside the MT5 and MQL5 ecosystem. The framework is designed around one shared execution engine, one shared data pipeline, and a plugin system that allows many model families to be compared under the same market conditions, labeling rules, and risk controls. Its purpose is practical: produce smarter `BUY / SELL / SKIP` decisions that survive realistic spreads, commissions, and changing intraday market regimes.

Why FXAI Exists
---------------
Most MT5 model projects collapse into one of two weak patterns: a single hard-coded strategy with limited extensibility, or a loose collection of disconnected experiments that cannot be compared fairly. FXAI solves that by standardizing feature generation, normalization, regime handling, warmup, online training, calibration, ensemble logic, and compliance checks behind one framework. This makes research faster, results more reproducible, and production deployment less fragile.

What The Framework Does
-----------------------
- Builds a shared M1-centric market dataset with multi-timeframe and cross-symbol context.
- Generates engineered features, applies configurable normalization, and routes the result into model plugins.
- Trains and updates plugins online with cost-aware 3-class labels: `SELL`, `BUY`, or `SKIP`.
- Evaluates plugin outputs through calibration, reliability handling, ensemble logic, and execution filters.
- Applies strict risk controls at the trade and equity level for realistic backtests and live operation.

Why It Is Useful
----------------
- One framework can benchmark many model families under the same rules.
- The data and labeling pipeline is centralized, which reduces hidden leakage and inconsistent experiments.
- Native MT5 execution keeps backtesting, optimization, and deployment in the same runtime environment.
- Cost-aware skip logic is built into the prediction target instead of bolted on after the model fires.
- The project remains pure MQL5: no DLLs, no external Python service, no external inference engine.

Framework Strengths
-------------------
- API-driven plugin architecture with explicit manifests, context, and prediction contracts.
- Shared feature and normalization pipeline across all plugins.
- Online learning and replay support for adaptive intraday models.
- Multi-model routing, ensemble stacking, and compliance validation inside the same engine.
- MT5-native workflow suitable for large Strategy Tester and MQL5 Cloud Network runs.

Plugin Naming Convention
------------------------
FXAI uses short category prefixes so the plugin name immediately shows what kind of model it is.

- `ai_*`  : neural and sequence-oriented models
- `lin_*` : online linear and sparse learners
- `tree_*`: tree and boosting models
- `dist_*`: distributional and probabilistic models
- `mix_*` : mixture-of-experts and uncertainty-routing models
- `wm_*`  : world-model and structural market models
- `mem_*` : retrieval and memory-based models
- `rule_*`: deterministic trading filters

Data Pipeline Summary
---------------------
The framework builds features from the current trading symbol and aligned context series using past-only data. The feature set covers short-horizon returns, candle geometry, trend filters, multi-timeframe structure, volatility and range measures, cross-symbol context, and time-of-day execution context. This allows both simple and complex plugins to learn from the same clean, comparable market representation.

Feature Groups
--------------
- Price structure and short-horizon returns
- Trend and multi-timeframe structure
- Volatility and range regime
- Cross-symbol market context
- Time and execution-friction context

Normalization Families
----------------------
- Market-structure-aware scaling
- Volatility-unit scaling
- Statistical standardization and robust scaling
- Relative-change and directional encodings
- Adaptive deep-model normalization

How FXAI Tries To Improve Real Trading Outcomes
-----------------------------------------------
The framework does not optimize for raw directional hit rate alone. It optimizes for trade quality under cost, uncertainty, and regime change by combining cost-aware labels, skip decisions, expected-move estimation, calibration, and execution filtering. That is the correct objective for short-horizon FX work where many mathematically correct forecasts are still economically useless.

Operational Model
-----------------
The MT5 Experts folder is the source of truth for the runnable system. The Git repository is the synchronized versioned copy that should only be updated after the active MT5 project compiles cleanly. This keeps the research loop disciplined and ensures the code that gets versioned is the same code that produced the tested `ex5`.

Bottom Line
-----------
FXAI is not a single trading strategy. It is a professional MT5 research and execution framework for systematic FX prediction, designed to compare model families fairly, train them under realistic trading constraints, and keep the full workflow inside MetaTrader 5.
